# 销售模拟演练系统推广方案

> 版本：v3.0（激进版）  
> 最后更新：2026-03-04  
> 状态：试点准备阶段  
> 场景：toC 车辆交付过程中的附加产品销售

---

## 一、目标设定（S.M.A.R.T. 原则）

### 1.1 目标设定逻辑

本方案采用 **三层目标体系**，从短期落地到长期价值，逐层递进：

```
近期目标（1个月）→ 中期目标（2-3个月）→ 长期目标（6个月）
      ↓                    ↓                    ↓
   融入关键触点        话术效果验证         能力成长平台
```

### 1.2 S.M.A.R.T. 目标定义

#### 🎯 近期目标（1个月）

**目标陈述**：把演练融入销售的关键工作触点，让销售在自然场景中完成演练而不感到额外负担

**场景说明**：
- toC 销售场景，车辆在交付过程中进行附加产品销售
- 客户在销售前不知道会被销售附加产品
- **无客户拜访前场景**，不占用晨会/夕会时间

**嵌入触点**（精简版）：

| 触点 | 触发时机 | 演练内容 | 频次 |
|------|---------|---------|------|
| 新人入职 | 入职第1周 | 基础话术 + 产品知识 | 3次必修 |
| 集中培训 | 培训期间 | 产品卖点 + 异议处理 | 1次集中演练 |
| 正式上岗前 | 上岗前1-2天 | 场景模拟 + 通关考核 | 1次通关 |
| 周例会/能力复盘前 | 每周/每两周 | 薄弱环节 + 优秀案例复盘 | 1次/周或1次/2周 |

| S.M.A.R.T. 维度 | 具体定义 |
|----------------|---------|
| **S - Specific（具体的）** | 嵌入4个关键触点：新人入职、集中培训、上岗前、周例会/复盘前 |
| **M - Measurable（可衡量的）** | 新人演练完成率 100%，上岗通关率 ≥ 80%，周活跃率 ≥ 60% |
| **A - Achievable（可实现的）** | 试点2个门店，每店5-6人（共10-12人），演练不占用额外工作时间 |
| **R - Relevant（相关的）** | 直接支持车辆交付场景中的附加产品销售，聚焦产品焦虑和购买意愿唤醒 |
| **T - Time-bound（有时限的）** | 第1周启动 + 培训，第2周上岗通关，第4周试点总结 |

**关键结果（KR）**：
- KR1：新人入职第1周完成3次基础话术演练（100% 完成率）
- KR2：上岗前通关考核通过率 ≥ 80%
- KR3：周活跃率 ≥ 60%（非新人销售每周至少1次演练）
- KR4：收集有效反馈 ≥ 15条，产品迭代 ≥ 2次

---

#### 📊 中期目标（2-3个月）

**目标陈述**：验证演练对话术改进的影响，尤其是产品焦虑和购买意愿唤醒话术的效果

**核心验证方向**：
- 演练前后话术对比：同一销售演练前后的话术变化
- 客户异议回答改进：模拟客户异议场景的回答质量提升
- 话术重点：**产品焦虑唤醒** + **购买意愿唤醒**

| S.M.A.R.T. 维度 | 具体定义 |
|----------------|---------|
| **S - Specific（具体的）** | 验证演练对销售话术的影响，重点关注产品焦虑和购买意愿唤醒话术 |
| **M - Measurable（可衡量的）** | 话术改进率 ≥ 30%（演练后 vs 演练前），异议处理得分提升 ≥ 20% |
| **A - Achievable（可实现的）** | 扩展至5-8家门店，积累50-80人样本 |
| **R - Relevant（相关的）** | 直接支持一线销售话术优化，让销售"越练越会卖" |
| **T - Time-bound（有时限的）** | 第6周话术对比报告，第10周效果白皮书 v1.0 |

**关键结果（KR）**：
- KR1：销售演练后话术 vs 演练前，关键话术采用率提升 ≥ 30%
- KR2：客户异议处理场景得分提升 ≥ 20%（前后对比）
- KR3：产品焦虑唤醒话术使用率 ≥ 70%（演练后销售）
- KR4：购买意愿唤醒话术使用率 ≥ 60%（演练后销售）
- KR5：输出《话术效果验证报告》，含前后对比数据

---

#### 🚀 长期目标（6个月）

**目标陈述**：用模拟演练指导销售能力成长，从"培训工具"升级为"能力发展平台"

| S.M.A.R.T. 维度 | 具体定义 |
|----------------|---------|
| **S - Specific（具体的）** | 建立个人能力画像、个性化推荐、成长路径、认证体系 |
| **M - Measurable（可衡量的）** | 销售主动查看"能力成长报告"比例 ≥ 50%，认证通过率 ≥ 70% |
| **A - Achievable（可实现的）** | 全国门店覆盖，系统与培训/晋升打通 |
| **R - Relevant（相关的）** | 支持销售全生命周期发展，成为能力评估客观依据 |
| **T - Time-bound（有时限的）** | Q1-Q2系统完善，Q3认证体系上线，Q4年度能力报告 |

**关键结果（KR）**：
- KR1：个人能力画像覆盖率 100%（所有活跃销售）
- KR2：AI 个性化推荐点击率 ≥ 40%
- KR3：认证体系参与率 ≥ 60%，通过率 ≥ 70%
- KR4：演练成绩纳入晋升评估体系（权重 ≥ 10%）

## 二、阶段推进与里程碑

### 2.1 阶段一：小范围试点（天使店，2个门店）

**时间**：4周  
**门店数**：2家  
**人数**：每店5-6人，共10-12人  
**目标**：验证可行性，达成近期目标

#### 阶段达成标志

| 维度 | 达标标准 | 验证方式 |
|------|---------|---------|
| **流程跑通** | 4个嵌入触点全部落地执行 | 系统数据 + 现场观察 |
| **用户接受** | 周活跃率 ≥ 60%，NPS ≥ 20 | 系统埋点 + 问卷调查 |
| **上岗通关** | 新人上岗前通关率 ≥ 80% | 考核记录 |
| **产品稳定** | 严重Bug数 = 0，一般Bug修复<48小时 | 问题追踪系统 |

#### 重要里程碑

```
第1周 ──→ 试点启动 + 产品培训 + 新人基础演练（3次）
   │
第2周 ──→ 上岗前通关考核 + 第1次周例会演练
   │
第3周 ──→ 第2次周例会演练 + 中期反馈收集
   │
第4周 ──→ 第3次周例会演练 + 试点总结 + 推广决策建议
```

### 3.3 协作配合模式

#### 周节奏协作

```
周一 ──→ 产品经理发布本周推荐演练包（聚焦产品焦虑/购买意愿）
         │
         ↓
周三/四 ─→ 门店经理组织周例会演练（15-20分钟）
         │
         ↓
周五 ──→ 培训负责人更新话术库（基于本周反馈）
         │
         ↓
周日 ──→ 系统自动生成个人/团队周报
```

#### 月节奏协作

```
第1周 ──→ 门店经理评审上月数据 + 问题汇总
          │
          ↓
第2周 ──→ 产品经理迭代功能 + 话术优化
          │
          ↓
第3周 ──→ 培训负责人更新内容 + 认证准备
          │
          ↓
第4周 ──→ 月度总结 + 下月计划
```

#### 阶段节奏协作


### 4.4 产品迭代节奏

```
试点期（第1-4周）
├─ 第1周：P0功能上线（短平快 + 基础话术 + 通关考核 + 即时反馈 + 基础看板）
├─ 第2周：问题修复 + 体验优化
├─ 第3周：基于反馈迭代v1.1
├─ 第4周：试点总结 + P1规划确认

规模期（第5-12周）
├─ 第5-6周：P1功能开发（焦虑唤醒 + 意愿唤醒 + 话术对比）
├─ 第7-8周：P1功能灰度测试 + 话术对比数据采集（第1次）
├─ 第9-10周：P1功能全量上线 + 话术对比数据采集（第2次）
├─ 第11周：话术效果白皮书v1.0
└─ 第12周：规模验证总结 + P2规划确认

全员期（第13-36周）
├─ Q2（第13-20周）：P2核心功能开发（AI动态对话 + 游戏化 + 认证体系）
├─ Q3（第21-28周）：P2功能上线 + 认证执行
├─ Q4（第29-36周）：系统打通 + 年度能力报告
└─ 持续：月度小迭代，季度大版本
```

### 4.5 给产品经理的特别提醒

| 注意维度 | 具体建议 |
|---------|---------|
| **不要过度设计** | 试点期功能越简单越好，核心是"能用 + 有用" |
| **场景聚焦** | toC车辆交付场景，聚焦产品焦虑和购买意愿唤醒 |
| **数据埋点先行** | 从第1天就埋好行为数据和话术数据，后续分析才有依据 |
| **反馈闭环要快** | 用户提的建议，48小时内必须有响应 |
| **话术迭代要快** | 一线反馈的话术问题，1周内必须更新话术库 |
| **别忽视"不好用"的声音** | 抵触情绪背后往往是真实痛点，深挖 |
| **与交付节奏对齐** | 车辆交付高峰前，提前准备对应演练内容 |

### 附录A：关键指标定义

| 指标 | 定义 | 计算公式 |
|------|------|---------|
| **周活跃率** | 当周至少完成1次演练的销售占比 | 活跃销售数 / 总销售数 × 100% |
| **新人演练完成率** | 新人第1周应完成3次演练的实际完成比例 | 实际完成数 / 3 × 100% |
| **上岗通关率** | 上岗前通关考核通过的销售占比 | 通过人数 / 考核人数 × 100% |
| **话术改进率** | 演练后关键话术采用率相比演练前的提升 | (演练后采用率 - 演练前采用率) / 演练前采用率 × 100% |
| **异议处理得分提升** | 异议处理场景演练得分的前后对比提升 | (后测得分 - 前测得分) / 前测得分 × 100% |
| **NPS** | 净推荐值，用户推荐意愿 | 推荐者% - 贬损者% |
